## ADDED Requirements

### Requirement: Vitest runs and passes a frontend smoke test
Vitest SHALL be configured (via `vitest.config.ts` or the Tauri template's existing config) and SHALL pass at minimum one unit test that renders the root `App` component using `@testing-library/react` without throwing.

#### Scenario: Smoke test passes
- **WHEN** the developer runs `pnpm test`
- **THEN** Vitest executes, at least one test passes, and the exit code is 0

#### Scenario: No test failures on clean checkout
- **WHEN** the repository is checked out fresh and `pnpm install && pnpm test` is run
- **THEN** all tests pass with no configuration required beyond `pnpm install`

---

### Requirement: Playwright runs and passes an E2E smoke test
Playwright SHALL be configured via `playwright.config.ts`. At minimum one E2E test SHALL launch the Vite dev server, navigate to `localhost`, and assert that the page title or a root element is present. The test SHALL pass with `pnpm test:e2e`.

#### Scenario: E2E smoke test passes
- **WHEN** the developer runs `pnpm test:e2e`
- **THEN** Playwright starts, the Vite dev server is launched (or reused), the page loads, and the test assertion passes with exit code 0

#### Scenario: Playwright browser binary is present
- **WHEN** `pnpm test:e2e` is run on a fresh checkout
- **THEN** the required Chromium binary is available (installed via `pnpm dlx playwright install --with-deps chromium`) and Playwright does not error on browser launch

---

### Requirement: cargo test runs and passes in the Rust crate
`cargo test` SHALL execute without build errors inside `src-tauri/` and SHALL pass at minimum one unit test (the scaffold default generated by `create tauri-app` is acceptable, e.g., a `greet` function unit test).

#### Scenario: Rust tests pass
- **WHEN** the developer runs `cargo test` inside `src-tauri/`
- **THEN** all tests pass with exit code 0 and no compilation errors

#### Scenario: Test suite runs in CI without extra tooling
- **WHEN** `cargo test` is run in an environment with the Rust toolchain (edition 2021, 1.77.2+) installed
- **THEN** the tests compile and execute without requiring additional system dependencies beyond the Rust toolchain

---

### Requirement: Test scripts are defined in package.json
The `package.json` SHALL define the following npm scripts so test commands are consistent across contributors and CI:
- `test` → runs Vitest in run mode (not watch)
- `test:e2e` → runs Playwright
- `test:watch` → runs Vitest in watch mode

#### Scenario: Scripts are present and executable
- **WHEN** `pnpm run --list` is executed
- **THEN** `test`, `test:e2e`, and `test:watch` appear in the output
